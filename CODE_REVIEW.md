# N-Verify Protocol - Code Review Checklist

## Before Submitting PR

- [ ] Code compiles without errors
- [ ] Tests pass
- [ ] No linting errors
- [ ] Types are correct
- [ ] No console.log left behind
- [ ] Error handling added
- [ ] Comments for complex logic
- [ ] Documentation updated if needed

## Reviewer Checklist

- [ ] Code is readable
- [ ] Logic is correct
- [ ] No security issues
- [ ] Performance is acceptable
- [ ] Tests are adequate
- [ ] Follows style guide

## Common Issues

- Missing error handling
- Hardcoded values
- Unused variables
- SQL injection risk
- XSS vulnerabilities
