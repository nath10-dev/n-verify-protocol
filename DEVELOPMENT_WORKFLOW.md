# N-Verify Protocol - Development Workflow

## Daily Routine

1. Pull latest changes
2. Create feature branch
3. Write code
4. Test locally
5. Commit changes
6. Push to remote
7. Create PR
8. Get review
9. Merge

## Branch Naming

- `feature/add-login`
- `fix/auth-bug`
- `docs/readme-update`
- `refactor/api-cleanup`

## Commit Messages

- `feat: add user login`
- `fix: resolve auth bug`
- `docs: update README`
- `refactor: clean API code`
- `test: add auth tests`

## Code Review

1. Check for bugs
2. Check for security
3. Check for performance
4. Check for readability
5. Check for tests
